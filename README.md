# Streaming Platform Demo

Proyecto Spring Boot con Builder y Singleton.

-Descripcion
En un centro poblado de la región atlántica de Colombia, un grupo de emprendedores culturales y audiovisuales ha identificado la necesidad de preservar, organizar y dar mayor visibilidad a las producciones cinematográficas independientes creadas en la zona. Actualmente, gran parte de las películas y series producidas por artistas locales se encuentran dispersas, almacenadas de forma informal o con muy poca difusión, lo que dificulta que la comunidad pueda acceder fácilmente a historias que representan su identidad, sus tradiciones y los acontecimientos más importantes de la región. Estas producciones incluyen relatos inspirados en hechos históricos sobresalientes, experiencias sociales de la población y narrativas con enfoques de fantasía, drama y cultura popular, las cuales poseen un importante valor artístico y cultural para las nuevas generaciones.
Ante esta situación, surge la necesidad de desarrollar una aplicación que permita digitalizar, administrar y gestionar de manera organizada todos los contenidos audiovisuales existentes y futuros del cine independiente local. La plataforma busca convertirse en un espacio centralizado donde los usuarios puedan consultar películas y series, clasificarlas por categorías y facilitar el acceso al contenido de manera sencilla y segura. Además, esta solución permitirá fortalecer la difusión cultural de la región, apoyar a los creadores audiovisuales y conservar el patrimonio narrativo de la comunidad, brindando una herramienta moderna y útil de fácil acceso. Por ende, la aplicación de administración de la plataforma Streaming desarrollada en Java con Spring Boot busca resolver la necesidad de administrar y visualizar contenido multimedia de manera organizada, escalable y centralizada, permitiendo gestionar películas y series categorizadas por distintos géneros, además de controlar el acceso de usuarios mediante autenticación con correo electrónico y contraseña. El sistema fue diseñado para ofrecer una arquitectura mantenible y preparada para futuras ampliaciones, como integración con bases de datos, seguridad avanzada o microservicios. En este contexto, el patrón Builder resulta clave porque facilita la construcción flexible y ordenada de objetos complejos como películas y series, permitiendo configurar múltiples atributos sin sobrecargar constructores y mejorando la legibilidad del código. Por otro lado, el patrón Singleton garantiza que exista una única instancia global encargada de administrar el catálogo multimedia, evitando inconsistencias de información y proporcionando acceso centralizado a los datos del sistema, lo que favorece la eficiencia, el desacoplamiento y la escalabilidad de la solución.

-Inicialmente se debe descargar y descomprimir el proyecto streaming-platform en una carpeta local del equipo. 
-Posteriormente, el proyecto puede abrirse desde un entorno de desarrollo como IntelliJ IDEA, Eclipse o Visual Studio Code, verificando previamente que se encuentre instalado Java 17 y Apache Maven.
-Una vez cargado el proyecto, desde la terminal ubicada en la raíz del sistema se debe ejecutar el comando mvn clean install para compilar las dependencias.
-Posteriormente se debe ejecutar el comando mvn spring-boot:run para iniciar el servidor Spring Boot. La aplicación quedará disponible en la URL: http://localhost:8080
-Para probar sus funcionalidades mediante herramientas como Postman, se utiliza el endpoint http://localhost:8080/api/auth/login mediante el método POST con las credenciales demo admin@streaming.com y 123456, mientras que el catálogo multimedia puede consultarse mediante el endpoint http://localhost:8080//api/catalog mediante el método GET, el cual retorna la información de películas y series en formato JSON.
